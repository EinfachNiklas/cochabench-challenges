# Third-Party Challenge Policy

This document defines the policy for challenges that are inspired by or derived from third-party material.

## Default Rule

Prefer original challenge content.

If a challenge is based on third-party material, it may only be included if the original source license allows the intended use.

## Allowed Sources

The safest sources are:

- Original work created for this repository
- Public domain material
- Material under permissive licenses such as MIT, BSD, or Apache-2.0, if their conditions are met

## Restricted or Disallowed Sources

Do not contribute challenges copied or adapted from:

- Repositories with no license
- Sources that forbid redistribution or commercial use
- Copyleft sources such as GPL, LGPL, or AGPL if inclusion would impose incompatible obligations on this repository
- Paid or proprietary course material, books, or platforms without explicit permission

## Required Contributor Disclosures

If a challenge uses third-party source material, the contributor must disclose:

- The original source
- The original license
- Whether the contribution is copied, adapted, or only inspired by the source
- Any attribution or notice requirements

## Attribution Requirements

When third-party material is included or closely adapted:

- Preserve required copyright notices
- Preserve required license notices
- Add clear attribution in the relevant challenge directory or accompanying documentation

If the original license requires including the full license text or notice text, that requirement must be met in the repository.

## Safe Patterns

Usually safe:

- Reusing an idea or problem domain while writing new challenge text, new starter code, and new tests from scratch
- Adapting material from permissive open-source projects with proper attribution and retained notices

Usually not safe:

- Copying challenge statements verbatim from another repository
- Copying tests or starter code from an unlicensed repository
- Importing GPL-family code or tests into this MIT-licensed repository

## Review Rule

Maintainers may reject any third-party-derived contribution if the provenance or license position is unclear.
