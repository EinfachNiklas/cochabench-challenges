# Contributing

Thanks for contributing to `cochabench-challenges`.

## Scope

This repository contains coding challenges for `cochabench`.
Contributions should fit the repository conventions documented in the root `README.md`.

## License of Contributions

By submitting a contribution to this repository, you represent that you have the right to submit it.

Unless explicitly agreed otherwise in writing by the repository maintainer, all contributions to this repository are provided under the MIT License of this repository.

Do not submit contributions under a separate or additional license.

## What You May Contribute

Examples of acceptable contributions:

- New challenges
- Improvements to existing challenges
- Better tests
- Better challenge descriptions
- Template improvements
- Metadata and release fixes

## Challenge Authoring Rules

When adding or updating a challenge:

- Start from the appropriate template in `template/`.
- Keep the challenge structure consistent with the repository conventions.
- Update `challenge.config.json` and `manifest.json` when needed.
- Keep challenge text and tests aligned with the intended evaluation setup.

## Third-Party Source Material

If your contribution is inspired by or derived from another project, repository, article, kata, or benchmark, you must follow the policy in [THIRD_PARTY_CHALLENGES.md](THIRD_PARTY_CHALLENGES.md).

Do not submit copied or adapted material unless you are sure the original license permits that use and all required notices are preserved.

## Pull Requests

For pull requests:

- Explain what you changed.
- Mention whether the change adds a new challenge or updates an existing one.
- Call out any third-party source material explicitly.
- Note any known limitations or follow-up work.

## Questions

If you are unsure whether source material can be reused or adapted, ask before submitting the contribution.
